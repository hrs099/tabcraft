# Project Skills

## Skill 1: Product Shell First
- Build the workspace before intelligence.
- Use mock JSON and demo projects before real model output.
- Validate correction and practice loops before adding analysis complexity.

## Skill 2: Trust-Centered UX
- Show confidence directly in workspace.
- Never hide uncertainty.
- Offer alternate suggestions for uncertain bars.
- Keep original AI output visible after user correction.
- Treat percussion markers with weak evidence as tentative.

## Skill 3: Guitar-First Logic
- Always consider tuning.
- Always consider capo.
- Remember the same pitch can map to many string/fret positions.
- Let user switch between easiest, closest, and original positions.

## Skill 4: Percussive Fingerstyle Positioning
- Frame the product as percussive fingerstyle-first.
- Keep V1 scope realistic with only basic percussion support.
- Show percussion markers in the workspace and timeline.
- Avoid claiming full percussion understanding too early.

## Skill 5: Practice-First Value
- Looping, slowdown, metronome, and speed trainer create real value even before perfect AI.
- Practice mode should remove clutter and maximize repetition speed.

## Skill 6: Scope Control
- V1 should solve solo fingerstyle and basic percussive workflow first.
- Advanced percussion intelligence and gesture understanding are later-stage systems.
- Avoid feature inflation before sync and correction feel solid.

# Problem and User

## Main problem

Learning from guitar video is slow because users must manually decode pitch, rhythm, string choice, fretboard position, and technique.

## User types

### Hobbiest Guitar learners

Needs synced tab, looping, slowdown, and clear replay.

### Transcription-focused guitarist

Needs accurate note mapping, timing, tuning, capo handling, and quick correction tools.

### Fingerstyle player

Needs string/fret detail, hand-position understanding, right-hand hints, and technique labels. Organised Player with creativity.

### Percussive acoustic player

Needs body-hit understanding, rhythm separation, and advanced gesture interpretation.

## Main pains

- Audio-only tools miss string choice and fret position.
- Manual tab writing takes too long.
- Video alone is hard to decode frame by frame.
- AI output cannot be trusted when uncertainty is hidden.
- Alternate tuning and capo can break fret mapping.
- Existing tools often help practice or notation, but not both in one guitar-first workspace.

## Main jobs to be done

- Upload a guitar video and get a first draft tab.
- Check where the AI is unsure.
- Use past performace as reference- how user solve similar situations in past and apply them on the current situation.
- Correct wrong notes fast.
- Replay exact bar and phrase sections.
- Practice difficult bars repeatedly.
- Understand how the part is physically played on guitar.
